```

P3（參考/備份）：

  - 文件更新
  - 回應時限：本週內
```

### 跨 Agent 協調 SOP
```
多 Agent 任務觸發條件：

協調流程：
  1. Amy 拆解任務為子任務
  2. 確認子任務依賴關係（串行 vs 並行）
  3. 下達指令並設定截止時間
  4. 各 agent 完成後向 Amy 匯報
  5. Amy 整合後向 Stanley 呈現
```

## 核心問題：「點知培訓有冇效？」

如果冇 KPI，培訓就係一場感覺良好嘅活動，唔係真正嘅升級。
每個 agent 必須對應至少一個可量化、有截止日期嘅成功指標。

## Iteration 3 總結

**最大發現：Toxic 有兩個「硬截止」KPI（WhatsApp 2026-06-01 / Apify 2026-05-31），呢兩個係整個系統嘅基礎，必須優先處理。**

**KPI Alignment 三個核心原則（全員明確）：**
1. **冇數字，唔算完成** — 每個任務必須有可量化嘅成功標準
2. **KPI 警報 = 義務升報** — 任何 agent 發現自己或別人嘅 KPI 警報，必須主動通知 Amy
3. **數字係鏡子，唔係武器** — KPI 用嚟改善，唔係用嚟懲罰，發現問題一起解決

*Iteration 3 完成 | 三輪主培訓 Loop 完成，進行 Bonus Round*

## 壓力測試場景：「黃金48小時」

**假設情境：** 痛症 IG 今日剛發布「局內人說話」系列首帖，突然爆紅，ER 達 18%，大量陌生人私訊。

### 系統正確反應流程

**T+0：帖子爆紅**
```
Leo → 🔴 緊急 Alert → Amy：
「帖子 ER 18%，遠超預期，建議立即 boost $200 擴大觸及」
Amy → Stanley：「Leo 建議 Boost $200，批准嗎？」
（Stanley 已說自動批准 → 繼續執行）
Amy → Toxic：「立即 boost 呢條帖，預算 $200」
```

**T+0 至 T+2：Lead 湧入**
```
Toxic WhatsApp 自動化啟動：
  自動歡迎訊息發送 → 詢問興趣類別
  分流：痛症 → 發送「局內人說話」系列介紹 + 免費諮詢預約連結
Toxic → Tony 交接：所有已確認有興趣嘅 lead 資訊包
Tony：2小時內逐個跟進
```

**T+2 至 T+24：內容趁勢**
```
Leo → Anna（直接）：
「呢個格式爆，建議今週再出一條同類格式，主題係 [香港人痛症地圖]」
Anna → Amy：「Leo 建議加急出一條，你同意我重新安排今週計劃？」
Amy：自動批准，Anna 即時起草第二條
```

**T+24 至 T+48：策略調整**
```
Small：分析爆紅原因
  → 「局內人角度 + 揭露行業真相 + 情緒共鳴 = 爆款公式」
  → 更新月度策略 Brief：「本月主打系列升為局內人說話，加快頻率至每週一條」
  → Anna 收到更新 Brief → 調整內容日曆
```

**T+48：Tony 情報回流**
```
Tony 週五摘要（提前提交）：
  「這48小時私訊50+，最常問：[具體問題]，最多拒絕原因：[原因]」
Small 收到 → 立即更新策略 → Anna 調整下一條帖方向
```

### 壓力測試結果

| 環節 | 升級前 | 升級後 | 改善 |
|------|--------|--------|------|
| Lead 響應速度 | 人工，可能隔天 | 即時自動 + 2小時人工跟進 | ⭐⭐⭐⭐⭐ |
| 競品/數據 alert | 等 Leo 想到才說 | 自動觸發即時通知 | ⭐⭐⭐⭐ |
| 內容趁勢速度 | 等 Stanley 指令 | Leo→Anna 直接協作 | ⭐⭐⭐⭐ |
| 策略→內容閉環 | 各做各 | Small Brief → Anna 執行 | ⭐⭐⭐⭐⭐ |
| 市場情報回流 | 零 | Tony 週五摘要→Small→Anna | ⭐⭐⭐⭐⭐ |

**壓力測試評級：PASS ✅**
升級後系統可以在 Stanley 唔介入嘅情況下，自主處理一個爆紅帖子帶來嘅所有後續動作。
